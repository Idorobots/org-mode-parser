* On Drawer Nesting and Custom Names

** The PROPERTIES Drawer

The =:PROPERTIES:= drawer is special — it holds node properties for the
heading it belongs to. It must directly follow the heading (or its planning
line) with no intervening blank lines.

** The Property Append Form

Node properties can be *appended* rather than replaced using the =NAME+= form.
This is useful for building up lists of values across multiple property entries.

* TODO Build feature extraction pipeline :pipeline:ml:
:PROPERTIES:
:tags: python
:tags+: sklearn
:tags+: spacy
:tags+: transformers
:related_papers: Vaswani2017
:related_papers+: Devlin2019
:related_papers+: Brown2020
:END:

The =tags= property above has the effective value =python sklearn spacy transformers=.
The =related_papers= property accumulates three values on separate =+= lines.

** Custom Named Drawers

Custom drawers (any name that is not =PROPERTIES=, =LOGBOOK=, or =END=) are
valid drawer elements. They are treated as opaque containers.

* The LOGBOOK Drawer (Conventional Name)

While =:LOGBOOK:= is just a custom drawer by name, Org Mode and many tools
treat it conventionally to hold state-change history and clock entries.
